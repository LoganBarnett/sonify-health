* TODO Pre-commit hook formats entire tree instead of staged files only
The =.cargo-husky/hooks/pre-commit= hook passes staged filenames as
positional arguments to =treefmt=, but treefmt may ignore positional arguments
and use its own tree walker.  This means =treefmt $STAGED_FILES= could format
every file in the repository, not just the staged ones.  Unstaged files get
reformatted on disk but are not re-staged, leaving dirty working tree
artifacts that have to be committed separately.

Forward-ported from rust-template (=420397e=).  The current treefmt version
shows =<paths>...= as a positional argument in =treefmt --help=, so the bug
may already be resolved.  Verify before implementing a fix.

Fix (if needed): use =treefmt --paths= (or the appropriate flag for limiting
treefmt to specific files) so only staged files are reformatted.  If treefmt
does not support per-file invocation, consider an alternative approach such as
running the language-specific formatters directly on the staged files.

* DONE Vibrato depth + rate
Added =vibrato_rate= and =vibrato_depth= voice parameters (commit =59c4e3d=).
Extended ranges: rate up to 200 Hz, depth up to 12 semitones for FM-like effects.

* DONE Drive, noise, crush, FM synthesis, downsample
Added six new voice parameters for gritty/textured sound:
- =drive= — pre-filter tanh saturation with normalized gain.
- =noise_mix= — pink noise blended before filter.
- =crush= — post-filter bitcrushing (exponential level mapping).
- =fm_ratio= — FM modulator frequency ratio.
- =fm_depth= — FM modulation index.
- =downsample= — sample rate reduction via hold_hz.
Also: rich sub-oscillator (weighted waveform mix instead of pure sine),
moog ladder filter replacing lowpass.

* DONE Add sustain parameter (ADSR envelope)
Add a =sustain= field to =Voice= (0.0–1.0, default 1.0) for a proper ADSR
envelope: Attack → Decay → Sustain (level) → Release.  Sustain = 1.0 behaves
like today (no decay stage).  Requires updating the envelope closure in
=heartbeat.rs= and =boop_graph=, the =VoiceOverrides= struct, CLI args, config
parsing, frontend, and all serialisation paths.

With lo/hi profiles the frequency should interpolate like every other voice

Consolidate: =BoopSpec= keeps =duration= only; frequency comes from the
