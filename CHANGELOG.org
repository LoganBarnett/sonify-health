* v0.4.0
** Added
1. A ~nix run .#quickstart~ app for anyone who has Nix: it runs the daemon
   against the Star Trek bridge preset with zero flags, laying down a writable
   ~config.toml~ in the working directory on first run so edits persist.
** Fixes
1. Linux release downloads now run on non-Nix hosts.  The published
   ~x86_64-linux-gnu~ and ~aarch64-linux-gnu~ binaries are portable dynamic
   glibc builds (~mkGnuPortablePackages~, zig against a glibc 2.17 floor) that
   link the host's ~libasound~, so they play through a desktop's PulseAudio or
   PipeWire; the static ~-musl~ builds remain the headless option.
* v0.3.0
* v0.2.0
** Added
1. A heavy sync with ~rust-template~ brings in a load of shared behavior as well
   as this very change log.
2. Releases now ship prebuilt ~sonify-health~ and ~sonify-health-server~
   binaries for Linux (glibc and static musl) and macOS (Intel and Apple
   Silicon), attached to each GitHub release with a SHA-256 digest and a
   build-provenance attestation.  A one-command ~curl ... | sh~ quick start
   (~install.sh~) downloads the latest release for the platform, lays down the
   Star Trek bridge preset, and runs it — a working, zero-configuration first
   run.  The README also documents the underlying ~curl~ download, ~cargo
   install~, and Nix routes.
** Fixes
1. The daemon now recovers from a silently stalled audio stream.  After a macOS
   output-device change that left the render callback un-pumped without a cpal
   error, playback would stay silent indefinitely; a liveness watchdog now
   detects the stall and rebuilds the stream on the current default device.
   Callback health is exposed via the new ~audio_seconds_since_last_callback~
   and ~audio_callback_stalls_total~ metrics.
