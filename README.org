* Introduction

sonify-health turns infrastructure health into ambient sound.  Each
machine self-reports via a heartbeat model: probe commands run
periodically, and their results select patches from a sound library.
Each heartbeat can contain multiple notes, and each note maps a metric
to patches via discrete thresholds or smooth gradients.

Heartbeats support three playback modes:

- *Clock* — one-shot plays on a wall-clock cycle, enabling
  time-slotted multi-machine ensembles.
- *Loop* — repeats a phrase with a configurable gap.
- *Continuous* — streams audio that crossfades on metric changes,
  useful for stress layers like CPU or memory pressure.

The result is a Star Trek bridge feel: rhythmic pulses and tonal
shifts that encode real meaning into peripheral awareness without
requiring visual attention.  Physical speaker placement gives spatial
location for free.

See [[file:sonification.org][sonification.org]] for the full design rationale.

* Installation

** From source (inside Nix devshell)

#+begin_src sh :results none :exports code
nix develop --command cargo build --release --workspace
#+end_src

The binary is at =target/release/sonify-health=.

** Nix build

#+begin_src sh :results none :exports code
nix build
#+end_src

The result symlink contains =bin/sonify-health=.

** Nix run (without installing)

#+begin_src sh :results none :exports code
nix run .#cli
#+end_src

The flake exposes several outputs for integration into NixOS and
nix-darwin configurations:

- =packages.${system}.default= — full workspace build.
- =packages.${system}.cli= — just the CLI/daemon binary.

* Usage

** Preview a patch

Play a named patch from the built-in library:

#+begin_src sh :results none :exports code
sonify-health preview --patch-name sine
sonify-health preview --patch-name alarm --freq 880
sonify-health preview --continuous --patch-name warm
#+end_src

Any patch parameter can be overridden on the command line.  Run
=sonify-health preview --help= for the full list.

** Print the patch library

Export the patch library in a paste-ready format:

#+begin_src sh :results none :exports code
sonify-health print --format toml
sonify-health print --format nix
sonify-health print --format cli --patch-name sine
#+end_src

** Run the daemon

Long-running mode that spawns probe commands, updates shared state,
plays heartbeats, and serves the web frontend:

#+begin_src sh :results none :exports code
sonify-health -c config.toml daemon
#+end_src

Stop with =SIGINT= (Ctrl-C) or =SIGTERM=.

Note that =--config= / =-c= is a global option and must appear
*before* the subcommand.

* Configuration

Copy =config.example.toml= to =config.toml=.  All fields are
optional; the daemon runs with sensible defaults.

Top-level keys:

- =log_level= / =log_format= — logging control.
- =listen= — web server bind address.
- =audio_device= — audio output device name (default: system default).
- =[patches]= — named patch definitions for the sound library.

Each =[[heartbeats]]= entry defines a probe and one or more notes.
The =playback= field selects the mode (=clock=, =loop=, or
=continuous=; defaults to =clock=).  The =result_mode= field
determines how the command output is read:

- =exit-code= — exit code 0/1/2 maps to discrete thresholds.
- =stdout= — command prints a metric value (0.0–1.0) to stdout.

Each note within a heartbeat carries a =transition= that maps the
probe metric to patches:

- =discrete= — selects the first state whose =threshold= exceeds the
  metric.
- =gradient= — interpolates between adjacent patches.

* Gotchas

- *Audio device access* — the daemon opens the default audio output.
  On Linux this requires ALSA or PipeWire.  On macOS, CoreAudio is
  used directly.
- *NTP precision* — multi-machine timing relies on wall-clock
  synchronization.  NTP provides ~10 ms accuracy, sufficient for
  the default cycle duration.
- *Nix sandbox* — =nix build= runs tests in a sandbox without audio
  hardware.  Integration tests that play sound are skipped in the
  sandbox build.
- *Global options* — flags like =-c=, =--log-level=, and =--listen=
  must appear before the subcommand, not after it.
