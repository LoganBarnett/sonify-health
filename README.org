#+title:     sonify-health
#+language:  en

* Introduction

sonify-health turns infrastructure health into ambient sound.  Each machine
self-reports via a two-layer audio model:

- *Heartbeat* — three percussive boops played in a time-slotted cycle.  Each
  boop encodes one subsystem check as healthy (baseline), degraded (minor third
  shift), or down (tritone shift).
- *Drone* — a continuous stress layer whose timbre, roughness, and volume shift
  in real time with system metrics like GPU load or memory pressure.

The result is a Star Trek bridge feel: rhythmic pulses and tonal shifts that
encode real meaning into peripheral awareness without requiring visual
attention.  Physical speaker placement gives spatial location for free.

See [[file:sonification.org][sonification.org]] for the full design rationale.

* Usage

** Preview sounds

*** Heartbeat

Play three boops at the given severity levels (0 = healthy, 1 = degraded,
2 = down):

#+begin_src sh :results none :exports code
sonify-health preview 0 0 0             # all healthy
sonify-health preview 0 1 0             # external connectivity degraded
sonify-health preview --heartbeat 2 2 2  # explicit --heartbeat flag
#+end_src

*** Drone

Play a continuous stress drone at a given metric value (0.0–1.0):

#+begin_src sh :results none :exports code
sonify-health preview --drone 0.8
sonify-health preview --drone --register low 0.5
sonify-health preview --drone --register high --duration 8 0.5
#+end_src

** Inspect a machine's voice

Print the hostname-derived voice parameters:

#+begin_src sh :results none :exports code
sonify-health voice
sonify-health voice --hostname silicon
#+end_src

** Run the daemon

Long-running mode that spawns check commands, updates shared state, and plays
the heartbeat at the configured time slot:

#+begin_src sh :results none :exports code
sonify-health daemon --config config.toml
#+end_src

Stop with =SIGINT= (Ctrl-C) or =SIGTERM=.

* Configuration

Copy =config.example.toml= to =config.toml=.  All fields are optional; the
daemon runs with sensible defaults (slot 0, 14-second cycle, no checks).

Key sections:

- =log_level= / =log_format= — logging control.
- =[voice]= — override any hostname-derived voice parameter.
- =[heartbeat]= — timing slot and check commands.

Each =[[heartbeat.checks]]= entry maps to one of the three boops.  The
=result_mode= field determines how the command result is read:

- =exit-code= — exit code 0/1/2 maps to healthy/degraded/down.
- =stdout= — command prints the severity value to stdout.

* Gotchas

- *Audio device access* — the daemon opens the default audio output.  On Linux
  this requires ALSA or PipeWire.  On macOS, CoreAudio is used directly.
- *NTP precision* — multi-machine timing relies on wall-clock synchronization.
  NTP provides ~10 ms accuracy, sufficient for 2-second slots.
- *Nix sandbox* — =nix build= runs tests in a sandbox without audio hardware.
  Integration tests that play sound are skipped in the sandbox build.
